This domain contains methods and templates for keeping stakeholders informed, aligned, and trained.

- **2.2.1-Newsletter**  
  Framework and template for recurring stakeholder updates (e.g. monthly or weekly product newsletters). Use this for consistent, scalable comms about progress, risks, and decisions.

- **2.2.2-Meeting-Agendas**  
  Agenda templates and frameworks for sprint reviews, retrospectives, planning sessions, stakeholder readouts, and other common PM meetings. Use this to run effective meetings that drive decisions.

- **2.2.3-One-Pagers**  
  Framework and templates for narrative one-pagers and lightweight briefs. Use this to align leadership and teams on proposals, decisions, and strategy.

- **2.2.4-Crisis-Management**  
  Comms playbooks and templates for when things go wrong (incidents, outages, serious bugs). Use this to communicate quickly and clearly during high-pressure situations.

- **2.2.5-Escalation**  
  Framework for knowing when and how to escalate issues in large organizations. Use this to navigate authority, resources, and decision-making effectively.

- **2.2.6-Saying-No**  
  Framework for declining requests gracefully without burning bridges. Use this when capacity is limited and you need to say no professionally.

- **2.2.7-Stakeholder-Management**  
  Framework for identifying, prioritizing, and managing stakeholders. Includes mapping, managing up, and cross-functional alignment guides.

- **2.2.9-Courses**  
  Course design framework and template. Use this to structure internal enablement, PM onboarding, or external training materials in a repeatable way.

Typical use: **Newsletter** for steady-state updates, **Meeting-Agendas/One-Pagers** for focused sessions and proposals, **Stakeholder-Management** for relationship building, **Escalation/Saying-No** for navigating requests, **Crisis-Management** for high-pressure situations, and **Courses** for longer-form learning.
